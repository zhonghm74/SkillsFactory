# Hypothesis Verification Loops

## Principle
Use answer-first, falsifiable hypotheses with predict-then-verify loops.

## Hypothesis Quality Bar
A hypothesis must be:
- decision-relevant
- specific
- testable/falsifiable
- prioritized

## Verification Pillars

### 1) Customer Structure
- Cohort analysis by acquisition source and behavior path.
- Verify whether metric shift is mix change vs behavior change.

### 2) Operational
- Event/dependency analysis across process chain.
- Use timestamp tracing and fault-tree style decomposition.

### 3) Policy/External
- Prefer DiD/control-group logic when possible.
- If no clean control exists, use counterfactual forecasting methods.

## Loop
Predict -> Test -> Compare expected vs observed -> Update confidence -> Decide next action.
