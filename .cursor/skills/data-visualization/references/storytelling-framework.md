# Data Storytelling Framework

## Core Concepts

### Big Idea
Define one sentence that captures the key takeaway.

Template:
- "The most important point is that [insight], which matters because [impact]."

### So-What Test
For every chart, answer:
- What decision should this change?
- What action should happen next?

## Narrative Arc (Progressive Disclosure)

1. **Setup** - Context and baseline.
2. **Conflict** - What changed or what is wrong.
3. **Insight** - Root cause or key relationship.
4. **Resolution** - Decision, recommendation, and next action.

## Annotation Patterns

Use concise annotation labels directly on charts:
- "Spike after campaign launch"
- "Drop caused by outage window"
- "Segment A overtakes Segment B"

## Delivery Template

1. Insight-led title
2. Supporting chart (clean, focused)
3. 2-4 annotations for key events
4. Action recommendation with expected effect

## Common Story Patterns

- **Change over time**: explain trend direction and turning points.
- **Intersections**: show when one segment overtakes another.
- **Drill-down**: start with aggregate, then inspect segment root causes.
- **Outliers**: investigate anomalies and implications.
